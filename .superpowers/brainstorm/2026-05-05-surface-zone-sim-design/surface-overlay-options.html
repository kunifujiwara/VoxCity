Building Surface Zone Marks On Simulation Results

Which marking style preserves the simulation colormap best while still making zones discoverable?

A. Low-opacity face tint

Easy to notice, but it mixes with the simulation colors and can hide exact values.

B. Boundary-only zone mark

Recommended: leaves face colors untouched and marks only the selected surface boundary.

C. Sparse hatch overlay

More visible than boundary-only, but still adds texture over the simulation result.

My recommendation: use B for simulation viewers, with A reserved only for the Zoning editor. For ground-level simulations, surface zones can use a slightly stronger translucent tint because they are context rather than the active data surface. For building-surface simulations, use boundary-only marks by default.
